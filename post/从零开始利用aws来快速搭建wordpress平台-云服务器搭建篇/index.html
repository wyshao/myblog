从零开始利用AWS来快速搭建WordPress平台---云服务器搭建篇

《开始利用AWS来快速搭建WordPress平台—准备篇》上一篇主要介绍了搭建平台 前的常用工具与AWS账号注册。接下来介绍在AWS EC2里创建实例和其他相关设置，从而搭建一个云服务器。
登陆AWS控制台后，会看到如下界面，找到“Launch a virtual machine”这一选项，并单击创建一个AMI（Amazon Machine Image）主机镜像。

接下来，完成云服务器的定制一共 七个步骤 ，这里分别来介绍
第一步 选择AMI
单击“Launch a virtual machine”之后，出现如下图所示的界面，这里注意要勾选左边的 “Free tier only”(仅免费套餐)。然后在右边的列表中会筛选出有 “Free tier eligible” (符合条件的免费套)字样的套餐 之后找到右边列表中的 Ubuntu Server 14.04 LTS (HVM), SSD Volume Type - ami-1450ad6b ，单击“Select”按钮进行选择，如下图所示。因为此选项用的人多，出问题也更好解决。 第二步 选择实例类型
这里就选择实例默认的类型“t2.micro”，注意下面要有“Free tier eligible“（符合条件的免费套餐）的字样，然后单击下一步去配置实例的详细信息，如下图所示。 第三步 配置实例
按系统默认选择即可，不用做任何修改，然后单击下一步进行存储配置 第四步 添加存储
这一步将添加存储配置信息，系统默认存储容量是8GiB，最大可选到30GiB。只要不超过30GiB就可免费。而卷类型选择“General Purpose SSD(GP2)“,如下图所示, 然后单击下一步 第五步 标签实例
这一步进行标签实例的命名，单击图中“click to add a name tag“,如下图所示 之后出现下图，随意自定义命名后，单击下一步进行安全组的配置 第六步 配置安全组
这一步要对安全组进行配置，这里要注意一下，创建一个新安全组，然后把常用的规则都添加进来，否则没法使用，其具体常用的规则配置如下图所示 第七步 审核
最后点击“Launch“(启动)，如下图所示， 然后会出现秘钥对（key pair）的配置，选择创建新密钥对，自行对新密钥对进行命名。此后，下载保存密钥对（.pem文件）并点击启动实例（Launch instances） 稍等片刻后，如果AMI创建成功，会出现如下界面，至此搭建好了一个AMI主机，实例下面的描述中有公有DNS和IP地址等信息。后面将在云主机上配置wordpress环境，如安装PHP, MySQL等。（这里为了保护隐私。抹去相关私人信息）
